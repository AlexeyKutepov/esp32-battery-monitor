# ESP32 Battery Monitor

Готовый пример системы мониторинга аккумуляторов на базе **ESP32 + INA219 + локальный сервер с веб-интерфейсом**.

## Что делает решение

- ESP32 измеряет напряжение аккумулятора через **INA219 по I2C**.
- При первом включении устройство открывает портал первичной настройки Wi‑Fi через **WiFiManager**.
- После подключения к сети устройство автоматически ищет сервер в локальной сети через **UDP broadcast discovery**.
- Устройство отправляет на сервер:
  - идентификатор устройства;
  - текущее имя устройства;
  - измеренное напряжение;
  - служебные данные (RSSI, IP, версия прошивки, счётчик пробуждений).
- После отправки ESP32 уходит в **deep sleep** и просыпается по таймеру.
- Сервер автоматически добавляет новые устройства в общий список.
- В веб-интерфейсе можно переименовать устройство.
- Новое имя хранится на сервере и передаётся устройству при следующем сеансе связи.

## Архитектура

### 1. Устройство ESP32

Скетч находится в файле:

- `firmware/esp32_battery_monitor/esp32_battery_monitor.ino`

Основной цикл работы:

1. ESP32 просыпается.
2. Считывает/восстанавливает настройки из `Preferences`:
   - `device_id`;
   - `device_name`.
3. Инициализирует INA219.
4. Измеряет напряжение аккумулятора.
5. Подключается к Wi‑Fi.
   - если сеть ещё не настроена, запускает captive portal `BatteryMonitorSetup`;
   - пользователь выбирает Wi‑Fi и вводит пароль.
6. Отправляет UDP broadcast-пакет `discover` на порт `4210`.
7. Сервер отвечает своим IP и HTTP-портом.
8. ESP32 делает `POST /api/devices/report` на сервер.
9. Сервер может вернуть `assigned_name`, если устройство было переименовано в веб-интерфейсе.
10. ESP32 сохраняет имя и уходит в deep sleep.

### 2. Сервер

Сервер расположен в папке:

- `server/app.py`

Он выполняет две роли:

- **UDP discovery listener** на порту `4210` — отвечает устройствам, где находится сервер;
- **HTTP API + веб-интерфейс** на порту `8080`.

Данные сохраняются в SQLite:

- `server/data/battery_monitor.db`

### 3. Веб-интерфейс

Фронтенд — обычная статическая страница без сборки:

- `server/static/index.html`
- `server/static/styles.css`
- `server/static/app.js`

Возможности:

- просмотр списка всех найденных устройств;
- просмотр последнего напряжения;
- просмотр времени последней связи;
- переименование устройства;
- автообновление списка каждые 15 секунд.

## API сервера

### `POST /api/devices/report`

Приём данных от устройства.

Пример JSON:

```json
{
  "device_id": "esp32-bat-ABCD12345678",
  "device_name": "Battery-12345678",
  "voltage": 12.41,
  "wifi_rssi": -58,
  "firmware_version": "1.0.0",
  "boot_count": 17,
  "uptime_ms": 3621,
  "ip_address": "192.168.1.55"
}
```

Ответ:

```json
{
  "status": "ok",
  "assigned_name": "Аккумулятор лодки",
  "server_time": "2026-03-22T12:00:00+00:00"
}
```

### `GET /api/devices`

Возвращает список устройств для веб-интерфейса.

### `PATCH /api/devices/<device_id>`

Меняет отображаемое имя устройства. Новое имя будет применено на самой ESP32 при следующем пробуждении.

## Что нужно для ESP32 в Arduino IDE

### Платы

Установить пакет **ESP32 by Espressif Systems**.

### Библиотеки

Установить через Library Manager:

- `Adafruit INA219`
- `Adafruit BusIO`
- `ArduinoJson`
- `WiFiManager` by tzapu

Стандартные библиотеки `WiFi`, `HTTPClient`, `Preferences`, `Wire` входят в платформу ESP32.

---

## Подключение INA219 к ESP32

По умолчанию в скетче используются пины:

- `SDA` → GPIO 21
- `SCL` → GPIO 22

Типовое подключение:

- `INA219 VCC` → `3.3V`
- `INA219 GND` → `GND`
- `INA219 SDA` → `GPIO21`
- `INA219 SCL` → `GPIO22`

Если у вас другие I2C-пины, поменяйте в скетче:

```cpp
const uint8_t kI2cSdaPin = 21;
const uint8_t kI2cSclPin = 22;
```

## Как загрузить прошивку

1. Открыть Arduino IDE.
2. Открыть файл:
   - `firmware/esp32_battery_monitor/esp32_battery_monitor.ino`
3. Установить нужную плату ESP32 и COM-порт.
4. Убедиться, что библиотеки установлены.
5. Нажать **Upload**.

### Поведение при первом запуске

После первой загрузки устройство:

- создаст точку настройки `BatteryMonitorSetup`;
- откроет captive portal Wi‑Fi Manager;
- после выбора домашней сети сохранит настройки;
- в дальнейших циклах будет подключаться автоматически.

## Как запустить сервер на Windows

Сервер сделан так, чтобы его можно было запустить с минимальной подготовкой.

### Вариант для пользователя Windows

В папке `server` есть два файла:

- `install_windows.bat` — создаёт виртуальное окружение и ставит зависимости;
- `start_windows.bat` — запускает сервер.

### Пошагово

1. Установить **Python 3.11+** с сайта python.org.
2. Открыть папку `server`.
3. Запустить `install_windows.bat`.
4. После установки запускать `start_windows.bat`.
5. Открыть браузер:

```text
http://localhost:8080
```

Если Windows Firewall спросит разрешение, нужно разрешить доступ для локальной сети, иначе ESP32 не сможет найти сервер.

## Как работает автообнаружение сервера

ESP32 не требует вручную прописывать IP сервера.

Механизм такой:

1. После подключения к Wi‑Fi устройство посылает UDP broadcast на порт `4210`.
2. Сервер слушает этот порт и отвечает своим адресом.
3. ESP32 получает IP и отправляет HTTP-запрос с данными.

Это удобно, если сервер запущен на обычном Windows‑ПК внутри домашней сети.

## Настройка интервала сна

В скетче интервал deep sleep задаётся константой:

```cpp
const uint64_t kSleepSeconds = 300;
```

Сейчас это **300 секунд = 5 минут**.

Можно поменять на любой другой интервал.

## Важное замечание по переименованию устройств

Поскольку ESP32 большую часть времени спит, сервер не может мгновенно «протолкнуть» новое имя на устройство.

Поэтому логика такая:

- вы меняете имя в веб-интерфейсе;
- сервер запоминает желаемое имя;
- при следующем пробуждении ESP32 получает его в ответе API;
- сохраняет локально и дальше использует уже новое имя.

Это самый надёжный вариант для low-power режима.
